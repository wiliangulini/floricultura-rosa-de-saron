# Melhoria de UI/UX

Solicitação: $ARGUMENTS

1. Leia o componente, layout, estilos, estados e testes relacionados.
2. Preserve Tailwind CSS v4, componentes existentes e comportamento funcional.
3. Priorize mobile, sem overflow horizontal, com hierarquia, contraste, foco, teclado e HTML semântico.
4. Preserve textos de negócio; no admin, mantenha interface e mensagens em português.
5. Não introduza biblioteca visual, tema global ou reescrita de tela sem autorização.
6. Verifique loading, erro, vazio, feedback e ações destrutivas.

Informe o que mudou visualmente, o comportamento preservado, validações e riscos. Não altere SEO ou regras de negócio como efeito colateral.
