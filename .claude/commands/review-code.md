# Revisão técnica

Escopo: $ARGUMENTS

Não altere arquivos.

1. Leia regras, diff, arquivos alterados, consumidores e testes relevantes.
2. Priorize bugs, regressões, segurança, contratos, dados e ausência de testes críticos.
3. Avalie a stack real e os fluxos de carrinho, WhatsApp, admin, autenticação, SEO e mobile quando afetados.
4. Não trate preferência estética como bloqueador.
5. Para cada achado, informe arquivo, evidência, impacto, severidade e correção objetiva.

Use severidade Crítico, Alto, Médio, Baixo ou Observação. Termine com: aprovado, aprovado com observações, requer ajustes ou bloqueado.
